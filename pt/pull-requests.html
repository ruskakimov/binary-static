Ganhe dinheiro com solicitações pull

Para incentivar a transparência e melhorar o desempenho e a funcionalidade do nosso site, a Binary.com disponibilizou em código aberto todo o front-end da nossa base de código e agora estamos incentivando ativamente a comunidade a contribuir ao seu desenvolvimento.

Pagaremos por qualquer projeto que for aceito e incorporado na base de código mestre.

Que projetos estão disponíveis?

Todas as plataformas e bibliotecas front-end da Binary.com são elegíveis.

Como envio uma solicitação pull?

  • Bifurcar o repositório Github relevante
  • Fazer alterações na sua bifurcação
  • Enviei uma solicitação pull ao mestre no repositório original do Github
  • Envie um e-mail para pull-requests@binary.com para pedir que a nossa equipe de desenvolvimento reveja a sua solicitação pull

Como faço para testar as minhas alterações?

Todos os projetos front-end da Binary.com são implantados usando o recurso de páginas Github. Por favor, familiarize-se com esse recurso e leia as instruções de implantação em cada repositório relevante.

Que tipos de alterações posso enviar?

  • Novos recursos
  • Melhores recursos de gráficos
  • Novos indicadores técnicos
  • Usabilidade melhorada
  • Compatibilidade de navegador melhorada
  • Melhor capacidade de resposta
  • Renderização rápida
  • Correções de bugs
  • Documentação melhorada
  • Melhor cobertura de testes

Posso criar uma nova interface de raiz?

Com certeza. Valorizamos projetos como novos aplicativos móveis, aplicativos de bots de conversação, extensões de navegador e novas ferramentas de gráficos.

Posso comunicar vulnerabilidades de segurança?

Sim, mas preferimos que você comunique vulnerabilidades de segurança através do nosso programa de bug bounty.

Posso contribuir ao código back-end?

O nosso código de back-end é principalmente Perl e as nossas contribuições de fonte aberta podem ser encontradas em CPAN. O código fonte correspondente pode ser encontrado nos nossos repositórios Github que começam com o prefixo "perl-". Agradecemos contribuições a esses repositórios, principalmente documentação melhorada e cobertura de testes.

Como serei pago?

Apenas pagamos por solicitações pull que foram aprovadas pela nossa equipe de desenvolvimento e incorporados no projeto mestre. Se a equipe de desenvolvimento aprovar a sua solicitação pull, lhe pagaremos em criptomoedas. Você pode escolher entre Bitcoin, Litecoin ou Ether.

Quanto vou ser pago?

A nossa equipe de desenvolvimento irá determinar o quanto você será pago. Você pode esperar o seguinte intervalo de pagamentos:

  • Alterações trivias – geralmente de $5 a $20
  • Mudanças pequenas - geralmente de $20 a $50
  • Alterações de tamanho médio – geralmente $50 a $250
  • Alterações médias – geralmente de $250 a $750
  • Grandes alterações – geralmente de $750 a $2.500
  • Alterações grandes/projetos inteiramente novos - $2.500 a $10.000

Eu mantenho a propriedade do código?

Não. Uma vez que a sua solicitação pull seja combinada com o mestre pela nossa equipe de desenvolvimento, e você tenha sido pago, você terá efetivamente entregue os seus direitos sobre o código. Ele depois se tornará parte da base de código mestre Binary.com sob licença de fonte aberta do repositório relevante.

Se as recompensas não forem previamente acordadas, como sei que serei pago de forma justa?

Queremos continuar a melhorar a nossa plataforma, portanto, está no nosso interesse pagar os nossos contribuidores de forma justa. Caso não o façamos, você irá parar de contribuir às nossas solicitações pull, o que tornará este projeto contraproducente.

Posso trabalhar para a Binary.com a tempo integral?

Estamos sempre à procura de talento – candidate-se através da nossa página carreiras.

Posso discutir as bases do código com a equipe de desenvolvimento Binary.com?

Com certeza! Envie um e-mail para pull-requests@binary.com para solicitar um convite para o nosso canal Slack.
