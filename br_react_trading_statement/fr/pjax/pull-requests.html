Soyez payé pour les demandes de tirage

Pour encourager la transparence et améliorer les performances et fonctionnalités de notre site Web, Binary.com a ouvert le code source de l’ensemble de son code front-end, et nous encourageons maintenant activement la communauté à contribuer à son développement.

Nous paierons pour toute demande de tirage qui est acceptée et fusionnée dans la base de code maître.

Quels projets sont disponibles ?

Toutes les plates-formes de front end Binary.com et librairies sont admissibles.

Comment puis-je soumettre une demande de tirage ?

  • Bifurquer le répertoire Github pertinent
  • Apportez vos modifications de code dans votre fourche
  • Soumettre une demande de tirage pour la version maître dans le répertoire GitHub original
  • Envoyer un email à pull-requests@binary.com pour demander à notre équipe de développement d'examiner votre demande de tirage

Comment puis je tester mes modifications ?

Tous les projets de front-end de Binary.com sont déployées à l’aide du système de pages Github. Veuillez vous familiariser avec cette installation et lisez les instructions de déploiement dans chaque référentiel pertinent.

Quel genre de modifications puis-je soumettre ?

  • Nouvelles fonctionnalités
  • Meilleur système de visualisation graphique
  • Nouveaux indicateurs techniques
  • Confort d'utilisation améliorée
  • Compatibilité navigateur améliorée
  • Meilleure réactivité
  • Rendu plus rapide
  • Corrections de bugs
  • Documentation améliorée
  • Meilleure couverture des tests

Puis je écrire un nouveau front-end à partir de zéro ?

Naturellement. Nous nous félicitons des projets tels que de nouvelles applications mobiles, applications chatbot, les extensions du navigateur et les nouveaux outils graphiques.

Puis-je soumettre des failles de sécurité ?

Oui, mais nous préférons que vous soumettiez des failles de sécurité par l’intermédiaire de notre programme de bprimes de bug.

Puis-je contribuer au code back-end ?

Notre code back-end est principalement en Perl, et nos contributions open source se trouvent sur CPAN. Le code source correspondant se trouvent dans notre Grépertoire Github qui commencent par le préfixe « perl- ». Nous saluons les contributions à ces répertoires, notamment ne matière de documentation améliorée et de couverture de test.

Comment serai-je payé ?

Nous payons seulement pour les demandes de tirage qui sont approuvées par notre équipe de développement et fusionnées dans le fichier maître. Si l’équipe de développement approuve votre demande de tirage, nous vous paierons en crypto devises. Vous pouvez choisir entre le Bitcoin, le Litecoin ou l’Éther.

A quoi puis-je m'attendre en matière de paiement?

Notre équipe de développement va déterminer combien vous êtes payé. Vous pouvez vous attendre à la gamme suivante de paiements :

  • Modifications triviales – généralement 5 à 20 dollars
  • Petits modifications – généralement 20-50 $
  • Changements de taille moyenne – généralement 50-250 $
  • Changements de taille moyenne – généralement 250-750 $
  • Changements importants – généralement $750-2500
  • Très grands changements/ projets entièrement nouveaux – $ 2 500-10 000

Est ce que je conserve la propriété du code ?

Non. Une fois votre demande de tirage est fusionnée dans la version maître par notre équipe de développement, et que vous avez été payé, vous avez effectivement renoncé à vos droits sur le code. Il deviendrait alors partie intégrante à la base de code maître Binary.com sous licence open source de la base de données pertinente.

Si les récompenses ne sont pas convenues à l’avance, comment puis-je savoir que je vais être payé assez ?

Nous voulons continuer à améliorer notre plateforme, donc il est dans notre intérêt de payer les contributeurs de manière équilibrée. Si nous ne le faisons pas, vous allez arrêter de contribuer aux demandes de tirage, ce qui rendrait ce projet vouée à l'échec.

Puis-je travailler pour Binary.com à temps plein ?

Nous sommes toujours à la recherche de talents – postulez via notre page carrières.

Puis-je discuter du code de base avec l’équipe de développement de Binary.com ?

Bien sûr ! S’il vous plaît envoyer un email à pull-requests@binary.com pour demander une invitation à notre chaîne Slack.
